AudioModding is a new thing in RocketLeague, and it cames with a lot of new concepts to learn and explore, having that in mind I've tried to write this documentation as complete and simple as possible for the ease of understanding of newcomers while keeping it non-redundant to prevent it from being a boring read for oldschool modders. Also, as the official documentation for the Wwise's UDK integration is no longer available anywhere (and it differs a lot from the UE4 integration), there may be undocumented features that I don't know nothing about.

Have to say that english is not my first language and I'm not a writer of any kind, at all. My english is self-taught so you should expect grammar mistakes along the documentation.

The documentation assumes you have basic knowledge about computers and UDK, and you're familiar with things such: System Explorer, Extensions, UE Packages and Asset types, among others.

IMPORTANT NOTE -- PLEASE READ

AudioModding works through custom .bnk/.wem files generated by the user. By default these files must be located in the -CookedPCConsole- folder in order to make RocketLeague able to load them, custom paths and subfoldering is not possible, this means it's not possible to read the audio files from the map's folder (wich usually isn't -CookedPCConsole-).

Luckily, if you're using BakkesMod, I've developed a Plugin named AudioLoader that forces the game to look for .bnk/.wem files on the folder where the map's .upk file is located.

With that said, have in mind that some users may not be using AudioLoader plugin and they'll need to move the files to their -CookedPCConsole- folder, so please try making their life easier by always packing your .wem files into the SoundBanks and naming your .bnk files with a unique name to prevent filename conflicts with other mods or the original RocketLeague files.

A good naming convention will be: #AUTHOR#_#MAP#.bnk, if you need more than one .bnk file, just append numbers at the end (E.G.: sopze_mymap.bnk, or sopze_mymap2.bnk).
