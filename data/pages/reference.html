WWISE PROJECT

The project aims to be a recreation of the one that Psyonix uses, everything has been built up manually from zero by decrypting the original init.bnk (a global bank file) and writing that data into a Wwise project. It has just one single purpose: To allow us generate SoundBanks that interact correctly with RocketLeague's audio system.

Do NOT rename anything that comes with the RocketLeague Wwise Project, otherwise it will stop working in-game. Also, changing anything inside any element that comes with the RocketLeague Wwise Project will have no effect in-game and may cause issues/crashes.

Now here's a brief explanation of the things you'll find in the RocketLeague Wwise Project:

PLACEHOLDER NAMES

Internally, Wwise don't use names, instead it convert names into numeric values using something called -fnv132 hash- upon generating SoundBanks, knowing that, the reason you'll find weird nonsense names in some element is because those names do actually convert into the same numeric value as the original name (wich has not been found yet), allowing us to reference and use that element correctly.

Sadly, placeholder names don't give us any clue about what an element really is for or what it does, this is especially noticeable (and annoying) when working with Game Parameters, but it's something we have to deal with by now.

MISSING PLUGINS

Elements using missing plugins have a placeholder plugin assigned onto the slot named with a -Placeholder__MissingPlugin- suffix to indicate it, and their ByPass option is ticked, I've added them like that just to know that the plugin is being used there. Note that having wrong names in this case doesn't matter as we wouldn't be able to use them anyway (cos missing plugin, you know).

However, you can actually use the missing plugins in-game, but only those that are already defined in ShareSets, just make sure to chose the one without the suffix and it'll be correctly assigned to your element in-game.
